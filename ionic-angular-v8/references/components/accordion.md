# ion-accordion

> Angular v8 standalone API reference. Generated from @ionic/core 4.43.0 docs.json.

- **Encapsulation:** shadow

## Import (standalone)

```ts
import { IonAccordion } from '@ionic/angular/standalone';
```

Add `IonAccordion` to the component's `imports: []` array.

## Properties

| Prop | Attribute | Type | Default | Description |
| --- | --- | --- | --- | --- |
| `disabled` | `disabled` | `boolean` | `false` | If `true`, the accordion cannot be interacted with. |
| `mode` | `mode` | `"ios" \| "md"` |  | The mode determines which platform styles to use. |
| `readonly` | `readonly` | `boolean` | `false` | If `true`, the accordion cannot be interacted with, but does not alter the opacity. |
| `toggleIcon` | `toggle-icon` | `string` | `chevronDown` | The toggle icon to use. This icon will be rotated when the accordion is expanded or collapsed. |
| `toggleIconSlot` | `toggle-icon-slot` | `"end" \| "start"` | `'end'` | The slot inside of `ion-item` to place the toggle icon. Defaults to `"end"`. |
| `value` | `value` | `string` | `ion-accordion-${accordionIds++}` | The value of the accordion. Defaults to an autogenerated value. |

## Slots

| Slot | Description |
| --- | --- |
| `content` | Content is placed below the header and is shown or hidden based on expanded state. |
| `header` | Content is placed at the top and is used to expand or collapse the accordion item. |

## CSS Shadow Parts

| Part | Description |
| --- | --- |
| `content` | The wrapper element for the content slot. |
| `expanded` | The expanded element. Can be used in combination with the `header` and `content` parts (i.e. `::part(header expanded)`). |
| `header` | The wrapper element for the header slot. |
